# Panduan Penggunaan Watermark PM — untuk Project Manager
(Wulan, Irham, Bastya)

Panduan ini menjelaskan cara menggunakan sistem Watermark PM secara dasar, khusus untuk peran Project Manager (PIC project). Tujuannya agar kalian paham alur dasarnya sebelum sosialisasi langsung.

---

## 1. Login

1. Buka link sistem (akan diberikan oleh admin).
2. Masukkan email dan password yang sudah didaftarkan.
3. Setelah login, kalian akan masuk ke halaman **Dashboard**.

---

## 2. Dashboard

Halaman ini menampilkan ringkasan umum:
- Total project, project aktif, win rate, project selesai.
- Posisi kas perusahaan (jika relevan).
- Pengajuan budget yang jatuh tempo dalam 14 hari ke depan.

Gunakan menu di bagian atas (navbar) untuk pindah ke halaman lain: **Projects, Workload Tim, Kalender, Nilai Tim, KPI, Finance, Target, Tim, Hutang**.

---

## 3. Membuat Project Baru

1. Buka menu **Projects**.
2. Klik **+ Project Baru**.
3. Isi data project:
   - Nama project, client, kategori (Meeting/Activation/Launching/dll).
   - Divisi (Event, Creative, atau PH).
   - PIC project (biasanya kalian sendiri jika kalian yang pegang).
   - Tanggal-tanggal penting: brief date, submit date, tanggal event/produksi.
   - Budget tier & kompleksitas event.
4. Klik **Simpan**.
5. Project baru otomatis muncul di daftar **Projects** dengan kode unik (misal P-034).

---

## 4. Mengelola Project — Halaman Detail Project

Klik nama project dari daftar **Projects** untuk masuk ke halaman detail. Ada beberapa tab:

### Tab "Tasks"
Ini adalah daftar pekerjaan/to-do untuk project tersebut.

- **Menambah task**: klik "+ Tambah task...", isi nama task, pilih siapa yang ditugaskan (assignee), prioritas (Low/Medium/High/Urgent), dan deadline (atau pilih "Open-ended" jika tidak ada deadline pasti).
- **Mengubah status task**: klik lingkaran bulat di kiri task. Urutannya: To Do → In Progress → Done. Klik lagi untuk mengembalikan ke To Do.
- **Task terkunci (Blocked)**: jika sebuah task tergantung pada task lain yang belum selesai, statusnya otomatis "Blocked" dan tidak bisa diklik sampai task yang ditunggu selesai.
- **Diskusi per task**: di bawah setiap task ada tombol **💬 Diskusi**. Klik untuk membuka kolom komentar — di sini tim bisa diskusi soal task tersebut.
  - Untuk menandai (tag) seseorang, ketik **@** lalu nama orangnya akan muncul pilihan, klik untuk pilih.
  - Orang yang ditag, dan orang yang ditugaskan (assignee) pada task tersebut, akan otomatis mendapat **notifikasi** (lonceng di pojok kanan atas).
- **Menghapus task**: klik ✕ di sebelah kanan task (hanya bisa oleh Project Manager).

### Tab "Tim"
Menampilkan PIC dan anggota tim yang terlibat di project ini. Project Manager bisa menambah/menghapus anggota tim di sini.

### Tab "Aktivitas"
Riwayat perubahan/aktivitas pada project (log otomatis).

### Tab "Info"
Detail lengkap project: client, tanggal, status pipeline (Hold → Pitching → Preparation → Event Day → Reporting → Invoicing → Done), hasil pitching (Win/Lose), dll. Bisa diedit oleh Project Manager.

### Tab "Penilaian Tim" (khusus PM/yang berhak menilai)
Tempat memberi nilai kinerja anggota tim untuk project ini (lihat bagian Nilai Tim di bawah).

---

## 5. Tugas Saya (My Tasks)

Setiap orang punya halaman **Tugas Saya** yang menampilkan:
- Semua task project yang ditugaskan ke kalian.
- To-do pribadi (bisa tambah sendiri, tidak terkait project).

Di sini juga ada fitur **Update Progress Harian** — setiap hari (sebelum jam 20:00), kalian wajib mengisi status progress untuk task yang sedang dikerjakan: **On Track / Delayed / Hold / Problem / Done** + catatan singkat.

⚠️ **Penting**: Update yang terlambat (setelah jam 20:00) atau status Delayed/Hold/Problem akan mempengaruhi nilai performa bulanan kalian. Jadi usahakan update tepat waktu setiap hari.

---

## 6. Nilai Tim (Penilaian Antar-Tim)

Menu **Nilai Tim** digunakan untuk menilai kinerja anggota tim yang terlibat dalam sebuah project:

1. Pilih project dari dropdown.
2. Untuk setiap anggota tim, isi nilai untuk beberapa kriteria (kontribusi terhadap project, kualitas eksekusi, kerjasama & komunikasi, dll) dengan skala 1-4 (Kurang/Cukup/Baik/Sangat Baik).
3. Klik **Simpan** per anggota.

Nilai ini bersifat **anonim** terhadap orang yang dinilai — mereka tidak tahu siapa yang memberi nilai, hanya melihat rata-rata skor mereka di halaman **Nilai Tim** masing-masing.

---

## 7. KPI

Menu **KPI** untuk penilaian KPI bulanan per divisi/role. Direktur/atasan menilai KPI anggota timnya setiap bulan berdasarkan kriteria yang sudah ditentukan per posisi.

---

## 8. Finance (Pengajuan Budget)

Jika project butuh dana (DP vendor, tiket, akomodasi, dll):

1. Masuk ke halaman **Finance** atau tab budget di project.
2. Ajukan pengajuan pembayaran: pilih kategori (tiket/transport, akomodasi, venue, vendor, talent, dll), jumlah, vendor/penerima, tanggal dibutuhkan.
3. Pengajuan akan masuk ke alur approval (Owner → Direktur → Direktur Finance → diproses Finance).
4. Status pengajuan bisa dipantau: Menunggu Approval → Disetujui → Sudah Dibayar.

---

## 9. Notifikasi

Ikon lonceng 🔔 di pojok kanan atas menampilkan notifikasi untuk kalian, misalnya:
- Ada task baru yang ditugaskan ke kalian.
- Ada yang menandai (tag) kalian di diskusi task.
- Ada komentar baru di task yang kalian pegang.
- Reminder pengisian KPI/nilai tim.

Klik notifikasi untuk langsung diarahkan ke halaman terkait.

---

## 10. Tips Umum

- Selalu cek **Tugas Saya** setiap pagi dan sore untuk update progress.
- Gunakan kolom **Diskusi** di setiap task agar komunikasi terkait pekerjaan tercatat rapi (tidak hilang di chat WA).
- Pastikan data project (tanggal, budget, PIC, anggota tim) selalu diisi lengkap agar laporan dashboard akurat.
- Jika ada kendala atau menemukan bug, catat dan laporkan ke admin sistem.
